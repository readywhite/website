# AI Estimate Rules

AI-assisted estimating should generate fast, package-based quote guidance while protecting margins and detecting exceptions that require review.

## Required Inputs

An estimate should reference:
- property address
- property type
- vacancy status
- timeline
- room count
- room size bands
- photos for each room
- worst-wall photo for each room
- ceiling/trim/add-on photos when applicable
- notes about stains, smoke, water damage, peeling paint, holes, and heavy prep

## Room Classification

Classify each room by:
- standard room
- large room
- small room
- hallway/common area
- ceiling included or excluded
- trim/doors included or excluded
- standard prep or exception prep

## Exception Detection

Flag for scope review when photos or notes show:
- severe prep
- water damage
- smoke damage
- stain blocking
- peeling paint
- wall repairs
- ceiling damage
- missing photos
- unclear room count
- occupied-property constraints
- accelerated timeline risk

## Quote Output Requirements

Every generated quote recommendation should include:
- package classification
- base price
- add-ons
- exception items
- assumptions
- photo gaps
- margin risk notes
- suggested GHL stage update
- recommended next customer message

## Margin Protection

Never let quote automation silently absorb exception work into the base package. Exceptions should either trigger review or be priced as separate add-ons/change orders.
