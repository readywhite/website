# Ready White Operating Context

This documentation tree is the operational memory scaffold for Ready White. It defines how the company standardizes fast-turn occupancy repainting, customer intake, estimating, vendor dispatch, automation, and production architecture.

## Operating Model
Ready White is a standardized property-turnover operating system using subcontractor fulfillment. The model depends on:
- Fast-turn occupancy repainting for vacant and turnover units.
- Standardized room pricing and package-based quoting.
- AI-assisted estimating from required room and damage photos.
- GoHighLevel-driven customer pipeline automation.
- Vendor dispatch workflows with scorecards and photo proof.
- Railway API backend for orchestration.
- GitHub as durable operational memory.

## Non-Goals For This Scaffold
This scaffold does not implement application logic, database schemas, workflow code, or external integrations. It only documents the operating model and creates stable locations for future SOPs, scripts, and standards.
