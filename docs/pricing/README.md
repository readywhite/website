# Pricing Context

## Objective
Create a standardized pricing system that enables fast quoting, protects gross margin, and supports recurring property-management work without custom estimating every vacancy.

## Pricing Principles
- Use standardized room pricing and package-based quoting.
- Quote from size bands, condition bands, room counts, and standard add-ons.
- Preserve speed-to-lead by minimizing manual price construction.
- Keep package pricing aligned with preset vendor buy rates.
- Escalate exception jobs before quote approval when scope risk is high.

## Standard Package Inputs
Document pricing tables here for:
- Room type.
- Room size band.
- Wall condition band.
- Ceiling inclusion.
- Trim inclusion.
- Door inclusion.
- Paint color changes.
- Unit-level minimums.
- Turnaround target.

## Standard Add-Ons
Track standardized add-ons for:
- Accent walls.
- Additional coats.
- Cabinet exclusions or referrals.
- Trim-heavy rooms.
- Stairways and hallways.
- Patch bundles.
- Stain-blocking primer.
- Smoke remediation prep.

## Exception Pricing
Hourly or change-order pricing is only allowed for:
- Severe prep.
- Water damage.
- Smoke or stain blocking.
- Custom repairs.
- Approved scope exceptions.

Every exception should document ROI impact, operational impact, scalability impact, and risk reduction.
