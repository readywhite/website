# Vendor Dispatch Workflow

Vendor dispatch begins after approval and scope verification. The workflow must preserve package standardization, response SLAs, photo proof, and operational redundancy.

## Canonical Flow

```txt
Approved
  -> Vendor Assignment
  -> Scheduled
  -> In Progress
  -> Photo Proof Review
  -> Completed
  -> Review Requested
  -> Closed Won
```

## Dispatch Inputs

Before assignment, confirm:
- approved package
- verified room or area count
- size bands
- add-ons
- exception items
- customer timeline
- property access details
- required completion deadline
- photo proof requirements

## Vendor Assignment Rules

Assign vendors using:
- preset package buy rates
- standard size bands
- standard add-ons
- vendor scorecards
- response SLAs
- capacity and geography
- callback history

Do not assign solely on hourly availability. Hourly/change-order work is reserved for approved exceptions.

## Vendor SLA Expectations

Track:
- response time to assignment
- schedule confirmation time
- on-time start
- on-time completion
- photo proof completion
- callback rate
- customer/property-manager feedback

## Photo Proof Requirements

Require vendors to submit completion photos before the job moves from `In Progress` to `Photo Proof Review` and then `Completed`.

Proof should include:
- wide completed room photos
- worst-wall completion photos
- add-on completion photos
- exception repair proof
- paint/material confirmation when needed

## Operational Redundancy

Maintain backup vendors for priority service areas and recurring property-manager accounts. If a vendor misses response SLA or schedule confirmation, escalate to backup assignment before the job threatens the turnover timeline.
