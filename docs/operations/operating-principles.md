# Ready White Operating Principles

Ready White is a standardized property-turnover operating system using subcontractor fulfillment. The goal is not to behave like a traditional painting contractor; the goal is to compress vacancy turnover time with repeatable quoting, scope verification, vendor dispatch, and CRM automation.

## Optimization Priorities

Optimize every change for:
- speed-to-lead
- operational reliability
- recurring property-management relationships
- standardized workflows
- reduced vacancy turnover time
- pipeline integrity
- fast execution
- scalable automation
- cash flow
- close rate
- recurring revenue
- operational leverage

Avoid vanity features that do not improve lead response, quote conversion, delivery speed, or repeat property-manager revenue.

## Standard Operating Vocabulary

Use:
- property refresh
- occupancy turnover
- scope verification
- vendor assignment
- photo proof
- package quote
- exception escalation

Avoid:
- handyman positioning
- luxury painter positioning
- bespoke artisan language
- default hourly labor framing

## Daily Systems Checks

Operational systems should be checked daily at:
- 00:00 EST
- 12:00 EST
- 18:00 EST

Checks should include:
- Railway `/health` status
- lead form submission path
- GoHighLevel API connectivity
- required Railway environment variables
- GHL pipeline and stage mapping integrity
- stale leads stuck without follow-up
- missed-call text-back health
- quote-sent follow-up status
- vendor assignment backlog
- photo proof review backlog

## Bottleneck Review Before Changes

Before implementing operational changes, identify whether the change affects:
- lead capture speed
- photo collection completion
- scope review time
- quote turnaround
- quote follow-up
- vendor availability
- schedule confirmation
- proof review
- review request timing
- close-out documentation

If a workflow can be standardized, automate it. If a workflow creates operational drift, constrain it.
